# Automation Department

## Mission
Design and maintain automated workflows that reduce manual effort and keep information flowing accurately between Success Brand's departments and tools.

## Scope
Covers internal workflow automation design and documentation. Does not include modifying core runtime code, deployment infrastructure, or the Astro website.

## Responsibilities
- Map manual processes suitable for automation
- Design workflow specifications for review
- Maintain a registry of automation candidates and status
- Document integration points between departments
- Monitor automation proposals for risk and reliability

## Inputs
- Process descriptions from other departments
- Tooling and integration constraints from Workspace
- Priorities from Michelle

## Outputs
- Workflow specifications and diagrams
- Automation backlog and status reports
- Documentation of integration requirements

## Agent Relationships
Reports to Michelle, who reports to Ju, who reports to the CEO. Supports all peer departments (Content, Sales, Research, Finance, Workspace, Mental Health, SuccessBrand) by documenting automation opportunities. Coordinates technical feasibility with the engineering team maintaining core/ and src/, which is outside this department's scope.

## Operating Rules
- Do not modify runtime code, core/, src/, or deployment files
- All proposed automations require human and Michelle review before implementation
- Document assumptions and risks for every proposal

## Daily Workflow
- Review the automation request backlog
- Draft or refine workflow specifications
- Log status updates in tasks.md
- Escalate technical feasibility questions to Michelle

## KPIs
- Number of documented automation opportunities (target TBD)
- Specification turnaround time (target TBD)

## Escalation Path
Automation agent escalates feasibility, risk, or scope questions to Michelle. Anything requiring changes to protected engineering areas (core/, src/, deployment, .github/) is escalated to Ju and then to the CEO and human leadership for coordination with the engineering team, never implemented directly.

## Current Status
Planned. This department has not yet been activated. No workflows, tools, or integrations are live. All processes described in this document are proposed designs pending approval and rollout.

## Known Limitations
- No automation platform or orchestration tool is connected yet
- No automation opportunities have been formally logged yet
- Dependency on the separate engineering team for any implementation is not yet formalized
- All KPI targets are placeholders pending human sign-off

## Future Roadmap
- Phase 1: Build an automation opportunity backlog
- Phase 2: Draft specifications for top candidates
- Phase 3: Coordinate pilot implementations with engineering
