# Workspace Department SOP

Status: Planned. This SOP describes the intended process for the Workspace department once activated. No step below is currently automated or live.

## Purpose
Define a repeatable process for keeping shared documentation organized, findable, and free of unnecessary duplication.

## Trigger
A new document is created, a department requests organization help, or a scheduled hygiene review comes due.

## Required Inputs
- Current folder and file structure
- Naming convention standards from Michelle
- Reports of duplicate or hard-to-find documents

## Step-by-Step Process
1. Review the relevant folder structure for the request
2. Identify organization issues (duplicates, unclear names, misplaced files)
3. Draft a proposed reorganization or naming fix
4. Submit the proposal to Michelle for approval
5. Apply approved, non-destructive changes only (for example, adding structure or renaming with owner consent)
6. Log any proposed deletions separately for explicit human approval
7. Update tasks.md and memory.md with outcomes

## Quality Checks
- No file is deleted or moved without approval
- Naming follows the agreed convention
- Structure changes do not break other departments' existing references without notice

## Escalation Rules
- Any proposed deletion: escalate to Michelle and a human before action
- Any request touching protected engineering folders: refuse and escalate to Michelle and Ju

## Handoff Rules
- Affected departments are notified before any structural change that touches their files
- Approved cleanup actions are logged and shared with Michelle

## Output Format
Proposal document listing current state, proposed state, and rationale, plus a change log entry once approved.

## Metrics
- Organization issues resolved (target TBD)
- Findability feedback score (target TBD)

## Failure Handling
If a proposed change might affect another department's work, the Workspace agent pauses, notifies the affected department and Michelle, and waits for confirmation before proceeding.
